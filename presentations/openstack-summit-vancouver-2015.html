Motivation

  • Not an open source solution available
  • Devs can built fault-tolerant apps

Devs don't consider...

  • Unexpected messages - not as in documentation
  • Server overloading - backoff responses
  • Link latency - different geolocation
  • Timeout
  • Connection drop

Defensive programming

  • Exception handling
  • Retry policies
  • Exponential backoff
  • Circuit breaker patterns
  • ...

HeliosBurn helps:

  • Monitor and record API traffic
  • Create the faults
  • test the resiliency against faults
  • validate the implementation

Key components

Proxy

  • Made in Twisted
  • Plugin architecture - Module pipeline
  • Modules must inherit from an abstract class
    • process_request, process_response
  • Filters requests and responses

API

  • Made in Django
  • Interacts with the Proxy via Redis
  • Manages the Proxy and modules

Current proxy modules

Traffic recoder

  • Records HTTP traffic
  • Export traffic to a Test Plan

Fault Injection

  • Alters traffic based on a set of rules
  • Rules define a filter and an action
    • Filter: condition to be met
    • Action: operation to be applied to the traffic

E.g.: Drop all traffic that contains "Firefox" in the User-Agent header

{
  "name": "firefox traffic drop",
  "description": "drops requests sent from firefox",
  "ruleType": "request",
  "filter": {
    "method": "GET",
    "headers": [{"key": "User-Agent", "values": ".*Firefox.*"}]
  },
  "action": {
    "type": "drop"
  }
}

QoS module

  • Link latency

Server overloading module
